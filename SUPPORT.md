# Support

FerroTeX is currently in an early design / pre-implementation phase.

## Getting Help

- Check the documentation index: `docs/README.md`
- If something is unclear or missing, open an issue once the repository is hosted on GitHub.

## Reporting Bugs

When reporting a bug in parsing or diagnostics, include:

- TeX engine and distribution (TeX Live / MiKTeX) and versions
- the command used to build (latexmk / tectonic / direct engine)
- the minimal `.tex` source to reproduce (if possible)
- the corresponding `.log` file (or a redacted excerpt)

## Feature Requests

Feature requests are welcome. For best results:

- state the user problem
- provide an example project or log snippet
- define expected behavior and acceptance criteria

## Security Issues

Do not report security issues publicly. See `SECURITY.md`.
